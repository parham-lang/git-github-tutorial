۱. دستوراتی که یاد گرفتم (Commands I learned)

در طول این تمرین، از دستورات زیر برای مدیریت پروژه استفاده کردم:





git config: برای تنظیم نام کاربری و ایمیل در سیستم.





git clone: برای کپی کردن مخزن از GitHub به کامپیوتر محلی.





git status: برای چک کردن وضعیت فایل‌ها در هر مرحله (یکی از پرکاربردترین دستورات).







git add: برای انتقال تغییرات به مرحله Stage (آماده‌سازی برای کامیت).







git commit: برای ثبت نهایی تغییرات در تاریخچه پروژه با پیام معنادار.





git log \& git log --oneline: برای مشاهده تاریخچه کامیت‌ها.





git branch \& git checkout: برای ایجاد شاخه‌های جدید و جابه‌جایی بین آن‌ها.





git stash \& git stash pop: برای ذخیره موقت تغییرات و بازگرداندن آن‌ها.







git revert: برای خنثی کردن یک کامیت خاص با حفظ تاریخچه.





git reset --soft: برای عقب گرد در کامیت‌ها در حالی که تغییرات فایل باقی بماند.







git rm --cached: برای حذف فایل از ردیابی Git بدون پاک شدن آن از روی سیستم.





git push: برای ارسال کامیت‌های محلی به مخزن آنلاین در GitHub.





git fetch \& git pull: برای دریافت و ادغام تغییرات از GitHub به سیستم محلی.







git diff: برای مشاهده تفاوت‌ها بین تغییرات یا کامیت‌های مختلف.







۲. چالش‌های روبرو شده (Challenges faced)

در این مسیر با چالش‌های زیر روبرو شدم که به من در یادگیری عمیق‌تر کمک کرد:



خطاهای سینتکسی در ترمینال: مانند استفاده از علامت‌های < > در دستور diff که متوجه شدم نباید تایپ شوند.





مدیریت Pull Request: در ابتدا درخواست ادغام را به مخزن اصلی فرستادم، اما یاد گرفتم چطور آن را در مخزن خود (Fork) مدیریت کنم.



تایپ دقیق نام فایل‌ها: تفاوت بین aboutme.txt و about-me.txt باعث شد بیشتر به جزئیات دقت کنم.



۳. نکات کلیدی (Key takeaways)

مهم‌ترین چیزهایی که یاد گرفتم:





اهمیت شاخه‌بندی (Branching): فهمیدم که کار روی ویژگی‌های جدید در شاخه‌های جداگانه چقدر به تمیز ماندن کد اصلی کمک می‌کند.





امنیت با .gitignore: یاد گرفتم فایل‌های حساس مثل پسوردها را هرگز نباید در گیت ردیابی کرد.







قدرت Pull Request: فرآیند همکاری در گیت‌هاب و نحوه ادغام کدها را به خوبی درک کردم.
